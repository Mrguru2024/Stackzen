# ✅ Updated StackZen Product & UX Criteria Checklist

## ✳️ Functional Criteria

- [x] **Visual Income Timeline**
  - [x] Timeline/chart for variable income is present
  - [x] Income events are color-coded and interactive
  - [x] Data is fetched and grouped by job/project
  - [ ] Weekly and monthly timeline toggle
  - [ ] Income pattern insight overlays (e.g. “Busy Weeks”)

- [x] **Smart Savings Automation**
  - [x] Users can set up savings rules based on cash flow
  - [ ] Automation triggers on spikes/dips
  - [ ] User is notified of automated savings actions
  - [ ] Saving trigger history log available
  - [ ] Pause/resume smart savings logic

- [x] **Job-Based Income Tracking**
  - [x] Users can tag/assign income to jobs/projects
  - [x] Dashboard summarizes income by job/project
  - [ ] Tag-based filters for export or insights
  - [ ] Integration with quote-to-paid workflows

- [x] **Project Forecasting Tools**
  - [x] Users can enter project details and expected income
  - [x] Forecasts are compared to actuals
  - [x] Alerts for forecast gaps
  - [ ] AI-based forecast suggestions
  - [ ] Missed forecast insights with Zen prompts

- [x] **Profit & Expense Dashboards**
  - [x] Real-time profit/loss and expense breakdowns
  - [x] Filterable by time, job, or category
  - [ ] Visual highlights for anomalies
  - [ ] Monthly trend indicators
  - [ ] Exportable profit reports

- [x] **Mentor Dashboard UI**
  - [x] Goal-tracking for mentees
  - [x] Resource sharing and interaction logs
  - [x] Mentor feedback and validation moments
  - [ ] Mentor availability and session scheduling
  - [ ] Stripe or booking integration panel

- [x] **Zen AI Assistant**
  - [x] Accepts user prompts in chat
  - [x] Responds with financial suggestions
  - [ ] Learns from tone/emotion matrix
  - [ ] Tracks user sentiment weekly
  - [ ] Personalized weekly summaries

## ✳️ UX Criteria

- [x] **Emotional State Cues**
  - [x] App detects or asks about user mood/state
  - [x] Supportive, context-aware messages are shown
  - [ ] Sentiment-informed UI changes (color, feedback)

- [x] **Guided Onboarding Paths**
  - [x] Onboarding adapts to user type (Gig, Creative, etc.)
  - [ ] User is routed to relevant questions/features
  - [ ] Onboarding feedback loop from Zen
  - [ ] Exit/resume onboarding at any step

- [x] **Mobile-First Simplicity**
  - [x] UI is thumb-friendly and easy to use on mobile
  - [x] Tap-based interactions and concise summaries
  - [ ] Offline access for income input/logs
  - [ ] Mobile notifications for income & goals

- [x] **Value-Centric Prompts**
  - [x] Prompts encourage reflection and planning
  - [x] No guilt/pressure language
  - [ ] Prompts adapt to financial behavior changes

- [x] **Safe-to-Spend View**
  - [x] Calming, clear "safe to spend" indicator
  - [x] Takes into account upcoming bills/goals
  - [ ] Safe-to-spend insights history view
  - [ ] Micro-adjustments suggested by Zen

---

## ✳️ Brand/Trust Criteria

- [x] **Human-First Tone**
  - [x] All copy is approachable and empathetic
  - [x] No cold financial jargon
  - [ ] Periodic tone reviews in product updates

- [x] **Mentor Validation Moments**
  - [x] Users receive mentor feedback and encouragement
  - [x] "You're on track" moments are surfaced
  - [ ] Milestone celebrations via Zen or UI badges

- [x] **Secure but Approachable Visuals**
  - [x] Modern, clean, but warm and friendly design
  - [x] Security cues (lock icons, privacy notes) are present
  - [ ] User-controlled privacy toggles for AI use
  - [ ] Transparent AI logging display (optional for advanced users)

- Use it for QA, dev readiness, PRD audits, and sprint validation.
